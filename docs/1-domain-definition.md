## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Todo List |
| 목적 | 인증된 사용자가 개인별로 할일을 카테고리 및 일정 기반으로 관리 |
| 핵심 문제 | 사용자 인증 기반의 개인 할일 관리 및 시작일/종료일 추적 기능 부재 |

## 2. 핵심 도메인 엔티티

### User (사용자)

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | UUID | Y | 고유 식별자 |
| email | String | Y | 로그인 이메일 (고유값) |
| password | String | Y | 암호화된 비밀번호 |
| name | String | Y | 사용자 이름 |
| theme | Enum | Y | UI 테마. `light` \| `dark` (기본값: `light`) |
| locale | Enum | Y | UI 언어. `ko` \| `en` (기본값: `ko`) |
| created_at | DateTime | Y | 가입일시 |
| updated_at | DateTime | Y | 수정일시 |

### Category (카테고리)

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | UUID | Y | 고유 식별자 |
| user_id | UUID | Y | 소유 사용자 |
| name | String | Y | 카테고리명 |
| is_default | Boolean | Y | 기본 카테고리 여부 (사용자당 1개만 true 허용) |
| created_at | DateTime | Y | 생성일시 |
| updated_at | DateTime | Y | 수정일시 |

### Todo (할일)

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | UUID | Y | 고유 식별자 |
| user_id | UUID | Y | 소유 사용자 |
| category_id | UUID | Y | 카테고리 (미지정 시 기본 카테고리 자동 적용) |
| title | String | Y | 할일 제목 (최대 200자) |
| description | String | N | 상세 내용 |
| is_done | Boolean | Y | 완료 처리 여부 (기본값: false) |
| start_date | Date | N | 시작일자 |
| end_date | Date | N | 종료일자 |
| created_at | DateTime | Y | 등록일시 |
| updated_at | DateTime | Y | 수정일시 |

> `status`는 DB에 저장하지 않으며, `is_done` / `start_date` / `end_date` / 오늘 날짜를 기반으로 조회 시점에 실시간 계산되는 파생값이다.

## 3. 비즈니스 규칙

| ID | 규칙 |
|----|------|
| BR-01 | 회원 가입 후 로그인한 인증된 사용자만 앱 사용 가능 |
| BR-02 | 사용자는 자신의 할일과 카테고리만 조회/수정/삭제 가능 |
| BR-03 | 카테고리를 지정하지 않으면 '기본' 카테고리가 자동 적용됨 |
| BR-04 | 각 사용자의 '기본' 카테고리는 삭제 및 이름 수정 불가 |
| BR-05 | `end_date`는 `start_date`와 같거나 이후여야 함 (`end_date >= start_date`) |
| BR-06 | `start_date` 또는 `end_date`가 없으면 status는 항상 `NOT_STARTED`로 간주 |
| BR-07 | 로그인한 사용자는 자신의 회원 정보(이름, 비밀번호)를 수정할 수 있음 |
| BR-08 | 테마 모드(`theme`)와 언어(`locale`)는 User 테이블에 저장되며, 로그인 응답에 포함되어 다음 접속 시 자동 적용된다 |
| BR-09 | 비로그인 상태에서는 브라우저 설정(`prefers-color-scheme`, `navigator.language`)을 기본값으로 사용한다 |

## 4. 할일 상태 정의

### 상태 목록 및 산출 조건

| 상태 | 코드 | 산출 조건 |
|------|------|----------|
| 시작 전 | `NOT_STARTED` | `is_done = false` AND `today < start_date` (또는 start_date 미설정) |
| 진행 중 | `IN_PROGRESS` | `is_done = false` AND `start_date <= today <= end_date` |
| 완료 | `DONE` | `is_done = true` |
| 기간 초과 | `OVERDUE` | `is_done = false` AND `today > end_date` |

> 날짜 경계 조건: `today = start_date`이면 `IN_PROGRESS`, `today = end_date`이면 `IN_PROGRESS`로 처리한다.

### 상태 전이

```
NOT_STARTED ──(시작일 도래)──► IN_PROGRESS ──(사용자 완료 처리)──► DONE
                                    │
                        (종료일 초과 & 미완료)
                                    ▼
                                 OVERDUE ──(사용자 완료 처리)──► DONE
```

## 6. 입력값 유효성 규칙

| 필드 | 규칙 |
|------|------|
| email | 이메일 형식 준수, 중복 불가 |
| password | 최소 8자, 영문+숫자 조합 필수 |
| name | 최소 1자, 최대 50자 |
| Category.name | 최소 1자, 최대 50자, 동일 사용자 내 중복 불가 |
| Todo.title | 최소 1자, 최대 200자 |
| end_date | `start_date`가 있을 경우 `end_date >= start_date` |
| theme | `light` 또는 `dark` 중 하나 |
| locale | `ko` 또는 `en` 중 하나 |

## 7. 용어 사전 (Glossary)

| 용어 | 정의 |
|------|------|
| 인증된 사용자 | 회원 가입 후 로그인하여 유효한 인증 토큰을 보유한 사용자 |
| 기본 카테고리 | 회원 가입 시 자동 생성되는 카테고리로 `is_default = true`. 삭제 및 이름 수정 불가 |
| 완료 처리 | 사용자가 명시적으로 할일을 완료 상태로 변경하는 행위 (`is_done = true`) |
| 기간 초과 | 종료일자가 오늘 날짜보다 이전이며 완료 처리되지 않은 상태 |
| status | DB에 저장되지 않으며 조회 시 `is_done`, `start_date`, `end_date`, 오늘 날짜로 계산되는 파생값 |
| theme | 사용자가 선택한 UI 테마. `light`(기본) 또는 `dark`. User 테이블에 저장되어 로그인 시 자동 복원 |
| locale | 사용자가 선택한 UI 언어. `ko`(기본) 또는 `en`. User 테이블에 저장되어 로그인 시 자동 복원 |
